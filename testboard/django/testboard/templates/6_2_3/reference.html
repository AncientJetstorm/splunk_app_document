    Generate An App Skeleton

    Open Splunk, navigate through your Splunk application to manage apps, click the button that says Create app
    Fill out the information and click on the save button.

    Or

    Open a terminal, cd to your Splunk home, cd under etc/apps/framework then run ./splunkdj createapp app_name and be sure to replace app_name with the name of your app.
    It should ask for your username and password, then it will create the app skeleton.

    Navigation Files

    In the $SPLUNK_HOME/etc/apps/app_name/default/data/ui/views/ folder, you should see a default.xml file that looks like this:



    This links Splunk to the file redirect.tmpl under the $SPLUNK_HOME/etc/apps/app_name/appserver/templates/ folder which goes to the home page.
    To add more links to your navigation bar, create a copy of default.xml then change the Home inside the <label> tags to what you want displayed, and change the redirect.tmpl inside the template attribute to a new name for the .tmpl
    Create a copy of the redirect.tmpl then change the Javascript area that says document.location to the correct location and save with the .tmpl file name that you refered to in the .xml file you created.
    Once those are created, go to the $SPLUNK_HOME/etc/apps/app_name/default/ui/nav/ folder and open the default.xml file, then add:

    <view name="xml_file_name" />

    And replace the xml_file_name with the name of the .xml file you created.
    You may also link directly to it like this:

    <a href="/dj/app_name/home">home</a>

    Just replace home with the file name.
    If you view the template_app(see the Template app section) then the default.xml file will have an example of each way for a page called color.



    There is also the <collection> tags that give you a dropdown list of links.

    Adding A Search Manager

    Django

    Open the HTML file you are working with, example home.html, then locate the {% block managers %} area and add:

    {% searchmanager id="search_id" search="splunk_search" cache=True %}

    Replace the search_id with the name you want to reference to, example pie_search_1, and replace the splunk_search with your search query.

    Javascript

    Open the HTML file you are working with, example home.html, then locate the // SEARCH MANAGERS area and add:

    var search_name = new SearchManager({
    "id": "search_id",
    "search": "splunk_search",
    "cancelOnUnload": true,
    "status_buckets": 0,
    "app": utils.getCurrentApp(),
    "preview": true,
    "runWhenTimeIsUndefined": false
    }, {tokens: true, tokenNamespace: "submitted"});

    Replace the search_name with what you want, example search1, then replace the search_id with the name you want to reference, example pie_search_1, and replace the splunk_search with your search query.
    If you have an input element effecting a search manager, then remove the , tokenNamespace: "submitted" from the bottom line. This allows the search to be updated.
    View the Element Attributes section for more things to add, example "earliest_time": "" and "latest_time": ""

    .table-chrome > thead > tr > th {
    background: #B33FF8;
    }
    // change the even numbered rows colors
    .table-striped > tbody > tr:nth-child(even) > td {
    background-color: #5384C7 !important;
    }
    // change the odd numbered rows colors
    .table-striped > tbody > tr:nth-child(odd) > td {

    }
    // change the table rows text colors
    .table-striped > tbody > tr > td {

    }

    Moving the Splunk JavaScript out of the HTML

    Copy and remove the Splunk script at the bottom of your HTML file and place in it it's own JS file.
    		At the top of the Javascript, replace this:

    require.config({

    baseUrl: "{{SPLUNK_WEB_PREFIX}}/static/js",
    waitSeconds: 0 // Disable require.js load timeout
    });

    With this:

    var urlprefix = document.URL.substr(0, document.URL.search("/dj"));

    require.config({

    baseUrl: urlprefix + "/static/js",
    waitSeconds: 0 // Disable require.js load timeout
    });


    ~~~~
